Diretivas AngularJS

Diretivas AngularJS são atributos HTML com a extensão ng-.
A diretiva ng-app inicializa uma aplicação AngularJS, já a diretiva ng-init inicializa dados da aplicação e a diretiva ng-model vincula o valor de um controle HTML (input, select, textarea) à um dado da aplicação.

Leitura sobre todas as diretivas AngularJS.

Exemplo utilizando as três diretivas

<div ng-app="" ng-init="firstName='Felipe'">

    <p>Name: <input type="text" ng-model="firstName"></p>
    <p>Você escreveu: {{ firstName }}</p>

</div>

Ver exemplo em funcionamento

A diretiva ng-app informa ao AngularJS que o elemento <div> é o "dono" da aplicação.

Data binding

A expressão {{ firstName }}, no exemplo acima, é uma expressão Angular data binding.
Em AngularJS data bindings vinculam uma expressão aos dados, ou seja, no exemplo, {{ firstName }} está vinculada com ng-mode=""firstName.

No próximo exemplo, dois campos de texto são carregados juntos com duas diretivas ng-model: 

<div ng-app="" ng-init="quantity=1;price=5">
    Quantity: <input type="number"  ng-model="quantity">
    Costs:    <input type="number" ng-model="price">

    Total em dólar: {{ quantity * price }}
</div>

Ver exemplo em funcionamento

Usar ng-init não é muito comum. Na sequência deste tutorial aprenderemos como criar dados no capítulo sobre controles.

Repetindo elementos HTML (looping)

A diretiva ng-repeat duplica, repete, clona elementos HTML, como veremos no exemplo abaixo: 

<div ng-app="" ng-init="names=['Jani','Hege','Kai']">
    <ul>
        <li ng-repeat="x in names">
            {{ x }}
        </li>
    </ul>
</div>

Ver exemplo em funcionamento

A diretiva ng-repeat clone os elementos HTML baseada na quantidade de itens da coleção.
Podemos usar a diretiva ng-repeat para acessar os itens de um array, conforme exemplo a seguir: 

<div ng-app="" ng-init="names=[
{name:'Jani',country:'Norway'},
{name:'Hege',country:'Sweden'},
{name:'Kai',country:'Denmark'}]">

    <ul>
        <li ng-repeat="x  in names">
            {{ x.name + ', ' + x.country }}
        </li>
    </ul>
</div>

Ver exemplo em funcionamento

A diretiva ng-app

A diretiva ng-app é o root da aplicação AngularJS e será inicializada automaticamente quando a página web for carregada.

A diretiva ng-init

A diretiva ng-init configura valores iniciais para a aplicação Angular.
Normalmente essa diretriz não será utilizada, utilizaremos controles e módulos.

A diretiva ng-model

A diretiva ng-model vincula o valor de controles HTML (input, select, textarea) à dados da aplicação.
A diretiva ng-model também pode:

  • Disponibilizar tipos de validação para dados da aplicação, como números, e-mail, campo obrigatório, entre outros;
  • Disponibilizar status para dados da aplicação dos tipos erro, dado inválido, entre outros;
  • Disponibilizar classes CSS "inline" nos elementos HTML;
  • Vincular elementos HTML à formulários HTML.

Nos próximos capítulos deste tutorial, serão apresentados mais detalhes sobre a diretiva ng-model.

Criando novas diretivas

Apesar do Angular possuir uma biblioteca de diretivas padrão, podemos construir nossas próprias diretivas.
Novas diretivas são criadas utilizando a função .directive.

Para realizar a chamada da nova diretiva, cria um elemento HTML como mesmo nome da diretiva (ou passe-o como atributo em um elemento HTML padrão).
Quando nomeamos uma diretiva, devemos usar camel case, ou seja, primeira palavra iniciada com letra minúscula e as demais sem separação com a primeira letra maiúscula apenas. No exemplo a seguir chamaremos a diretiva de: w3TestDirective.
Já nas chamadas da diretiva, escrevemos o nome inteiro com letras minúsculas e separamos as palavras por hífen. No exemplo em questão, ficaria escrito assim: <w3-test-directive>.

Exemplo:

<body ng-app="myApp">
    <w3-test-directive></w3-test-directive>

    <script>
        var app = angular.module("myApp", []);
        app.directive("w3TestDirective", function() {
            return {
                template : "<h1>Made by a directive!</h1>"
            };
        });
    </script>
</body>
