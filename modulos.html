Módulos AngularJS

Um módulo AngularJS marca - define como - uma aplicação.
O módulo é um "bloco" que pode comportar diferentes partes da aplicação e "guarda" os controles dela.
É válido dizer também que todo controle pertecendo à um módulo.

Criando um módulo em AngularJS

Um módulo é criado utilizando a função angular.module, conforme exemplo abaixo: 

<div ng-app="myApp">...</div>
<script>
    var app = angular.module("myApp", []);
</script>

O parâmetro "myApp" é passado como referência ao HTML informando em qual elemento a aplicação rodará.
A partir daí, já é possível adicionar controles, filtros, diretivas, validações e etc à nossa aplicação AngularJS.

Adicionando controles

Neste exemplo adicionaremos um controle à aplicação e o referenciaremos utilizando a diretiva ng-controller:

<div ng-app="myApp" ng-controller="myCtrl">
{{ firstName + " " + lastName }}
</div>

<script>
    var app = angular.module("myApp", []);

    app.controller("myCtrl", function($scope) {
        $scope.firstName = "Felipe";
        $scope.lastName = "Ranieri";
    });
</script>

Adicionando uma diretiva

O AngularJS possui um conjunto de diretivas internas (proprietárias) as quais podemos utilizar para incrementar funcionalidades à nossa aplicação.
Referência completa de diretivas AngularJS

Podemos usar módulos para adicionar nossas próprias diretivas à uma aplicação AngularJS, conforme exemplo a seguir: 

<div ng-app="myApp" w3-test-directive></div>

<script> 
var app = angular.module("myApp", []);

app.directive("w3TestDirective", function() {
    return {
        template : "Feito utilizando um construtor de diretivas"
    };
});
</script>

Ver exemplo em funcionamento

Uma diretiva pode ser chamada em diversos pontos da aplicação, conforme exemplo a seguir:
<div ng-app="myApp">
    <span w3-test-directive></span>
    <ul>
       <li w3-test-directive></li>
       <li w3-test-directive></li>
       <li w3-test-directive></li>
    </ul>
</div>
<script>
    var app = angular.module("myApp", []);
       app.directive("w3TestDirective", function() {
       return {
          template : "Feito utilizando um construtor de diretivas"
       };
    });
</script>

Módulos e controles em arquivos

É muito comum em AngularJS, as aplicações chamarem módulos e controles através de arquivos Javascript.
No exemplo a seguir, "myApps.js" contém a definição do módulo da aplicação, enquanto que o arquivo "Ctrl.js" contém o controle. 

<div ng-app="myApp" ng-controller="myCtrl">
{{ firstName + " " + lastName }}
</div>

<script src="myApp.js"></script>
<script src="myCtrl.js"></script>

No arquivo "myApp.js" temos a definição da aplicação através do seguinte código:
var app = angular.module("myApp", []);
Já no arquivo "Ctrl.js" definimos o controle através do código:
app.controller("myCtrl", function($scope) {
    $scope.firstName = "John";
    $scope.lastName= "Doe";
});

Nota:

O parâmetro "[]" na definição do módulo, pode ser utilizado para incluir dependências.
Sem este parâmetro, não estamos criando um novo módulo, sim recuperando um já existente.

Ver exemplo em funcionamento

Funções podem poluir o escopo global

Devemos evitar funções globais em arquivos Javascript, pois estas funções podem ser facilmente sobrescritas ou destruídas por outros scripts.
Módulos AngularJS reduzem esse problema mantendo todas as funções no módulo.

Quando carregar a biblioteca

Enquanto é comum nas aplicações HTML realizar a chamada dos scripts no final do <body>, em AngularJS é recomendado carregar a biblioteca tanto no elemento <head> como no começo do elemento <body>.
Esta prática é recomendada, por que a chamada do angular.module só é compilada após a carga da biblioteca.
